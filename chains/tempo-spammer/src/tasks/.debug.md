# Command Line to Debug Task

## Quick Debug (Single Task)

Use `tempo-debug` to debug a specific task for a specific wallet.

**From Project Root (Recommended):**
```powershell
# Debug Task 15 using Wallet 0 (Random Proxy)
cargo run --release --bin tempo-debug -- --task 15 --wallet 0

# Debug Task 15 using Wallet 0 (Specific Proxy 5)
cargo run --release --bin tempo-debug -- --task 15 --wallet 0 --proxy 5
$env:WALLET_PASSWORD="your_password"; cargo run --bin tempo-debug -- --task 16 --wallet 390

# Debug Task 15 using Wallet 0 (Direct Connection)
cargo run --release --bin tempo-debug -- --task 15 --wallet 0 --proxy 0
```



> **Note**: Always use `--release` for performance closer to production. Remove `--release` only if you need faster compilation for quick logic checks.

## Available Arguments

- `--task <ID>`: The task number to run (e.g. `1`, `15`, `30`)
- `--wallet <INDEX>`: The wallet index (0-based) from `wallets`.
- `--proxy <INDEX>`: 1-based proxy index. `0` for direct/no-proxy. Omit for random.
- `--no-db`: Skip logging to SQLite.

## Compilation

If you changed code, rebuild first:
```powershell
cargo build --release --bin tempo-debug
```

preparing up new wallet:
cargo run --release --bin tempo-sequence -- --workers 20
